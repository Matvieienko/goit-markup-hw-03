## Критерії приймання роботи наставником

### Проєкт

У корені проєкту є папка images з зображеннями
У корені проєкту є папка css з файлами стилів
Всі стилі написані у своїх відповідних файлах, які знаходиться в папці css та коректно підключені
У назвах файлів відсутні кирилиця, пробіли. Назви містять тільки літери, числа і слова англійської мови
Вихідний код відформатований за допомогою Prettier
Усі зображення та текстовий контент взяті з макета
Підключений нормалізатор стилів modern-normalize (мініфікована версія)
Код написаний з дотриманням **настанови**

### Організація CSS

1. В `index.html` підключається тільки `main.css`

2. `main.css` має включати лише імпорти стилів з інших файлів без інших стилів

3. У `main.css` імпортуються файли в такому порядку:

`@import './common.css';
@import './header.css';
@import './hero.css';
@import './features.css';
@import './team.css';
@import './portfolio.css';
@import './footer.css';`

4. Шляхи в `@import` мають починатися з ./

5. Глобальн і базові стилі описані в `common.css`, інші стилі, пов'язані зі стилізацією конкретних блоків сторінки (хедер, секції, футер тощо) описуються у відповідних CSS-файлах.

### Секція Portfolio

- На секції присутній `id`:
- `<section id="portfolio">`
- Розмітка секції Our Portfolio набрана у файлі index.html. Виконана HTML-розмітка всіх елементів макета
- Теги використані відповідно до їх семантичного змісту
- HTML проходить перевірку валідатором без помилок
- Імена класів описові та зрозумілі іншому розробнику
- Імена класів не містять великих літер, чисел, пробілів, трансліту і назв тегів. Імена містять тільки літери і слова англійської мови. Якщо ім'я класу складається з декількох слів, вони розділяються дефісом
- У тегів <img> вказані атрибути розмірів, як мінімум width
- Зображення експортовані з макета у форматі jpg
- Групи однотипних елементів зібрані у списки <ul>
- Всі необхідні шрифти за макетом та їх варіації (вага і накреслення) підключені з сервісу Google Fonts об’єднаним посиланням. Тобто в одному посиланні одночасно підключається і `Raleway`, і `Roboto`
- В хедері та футері логотип розділений тегом <span> і акцентована частина — “Web”.

### Оформлення

- Для оформлення використовуються селектори класу
- В стилях відсутній `!important`
- Інтерактивні елементи (кнопки і посилання), при наведенні мишкою або фокусі з клавіатури, мають активний стан, вказаний в Style guide (зміна кольору)
- Текст контактів в хедері змінює колір при ховері і фокусі
- Для елемента <body> задана властивість font-family з домінантним шрифтом на макеті Roboto
- Вказаний альтернативний варіанти сімейства шрифта (без зарубок) в кінці переліку font-family в елемента <body>
- Сімейство шрифтів Roboto явно задане тільки для елемента <body>, інші елементи наслідують його
- Для елемента <body> задана властивість color з домінантним на макеті кольором тексту
- Розмір шрифту (властивість font-size) всіх текстових елементів точно відповідає значенням з макета
- Висота рядка (властивість line-height) всіх текстових елементів точно відповідає значенням з макета і задана як множник, а не в px
- Колір (властивості color та `background-color`) всіх елементів точно відповідає значенням з макета
- Вага шрифту (властивість `font-weight`) всіх текстових елементів точно відповідає значенням з макета. Вага шрифту явно вказана тільки, якщо значення в макеті відрізняється від стандартного для цього елемента в браузері
- Кнопкам задана властивість cursor зі значенням pointer. «С14» У стилях не повторюються значення властивостей, які задані браузером за замовчуванням. Наприклад, посиланням не потрібно вказувати `cursor: pointer`, а абзацам `font-style: normal` або `font-weight: 400`
- Прихований заголовок <h2> “Our Features” не стилізований. В цій ДЗ його поки має бути видно, це ок. Приховування цього заголовка буде в наступній ДЗ.
